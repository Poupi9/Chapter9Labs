
# Practice Repository

This repository was created for Chapter 9 lab exercises.

## Author
Geoffrey Prevot

## Purpose
To practice shell commands, Git initialization, status checking, staging, and committing.

## Reflection
Git helps track changes without creating many duplicate filenames.

## Goal of this chapter:

We will build the Git workflow in small stages:

1. Check the terminal and confirm Git is installed  
2. Practice basic shell navigation  
3. Create the first project file  
4. Configure Git identity  
5. Initialize a local repository  
6. Check repository status  
7. Stage and commit the first file  
8. Create a second file and compare staged vs untracked changes  
9. Modify a tracked file and inspect the difference  
10. Review commit history  
11. Optionally connect the local repository to GitHub  

This step-by-step method helps you understand **what each command does** before combining them into a complete local Git workflow.
